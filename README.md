# 测试用例评审工具 Pro - 使用说明

## 📖 工具简介

**测试用例评审工具 Pro** 是一款基于浏览器的本地化测试用例管理工具，支持 Markdown 格式的测试用例文档导入、在线评审、标签管理以及多种格式导出功能。

## ✨ 主要功能

### 1. 文档管理
- **导入 Markdown 文件**：支持导入符合规范的 Markdown 格式测试用例文档
- **章节导航**：自动解析文档结构，提供快速定位功能
- **实时统计**：显示测试类别、测试点、优先级分布等统计信息

### 2. 评审功能
- **多条评审**：每个测试用例可添加多条评审意见
- **优先级标识**：自动识别并统计关键、高、中、低优先级用例
- **标签管理**：支持为测试用例添加自定义标签

### 3. 导出功能
提供多种导出选项：
- **📤 导出带评审**：导出包含评审意见的完整文档
- **📄 导出无评审**：导出原始测试用例（不含评审内容）
- **📋 导出评审文档**：单独导出评审意见汇总
- **📝 导出MD模板**：导出 Markdown 格式模板供参考

## 🚀 快速开始

### 步骤 1：打开工具
在浏览器中打开 `测试用例评审工具.html` 文件

### 步骤 2：导入文档
1. 点击 **📁 导入文件** 按钮
2. 选择符合格式的 Markdown 测试用例文档
3. 等待文档解析完成

### 步骤 3：查看内容
- 左侧显示 **章节导航**，点击可快速跳转
- 中间显示 **文档内容**
- 顶部显示 **统计信息**（测试类别、测试点数量、优先级分布等）

### 步骤 4：添加评审
1. 在需要评审的测试用例处进行标注或评论
2. 添加评审意见和标签
3. 系统自动统计评审数量

### 步骤 5：导出文档
根据需要选择相应的导出按钮：
- 需要保留评审内容 → **导出带评审**
- 仅需要测试用例 → **导出无评审**
- 单独查看评审意见 → **导出评审文档**

## 📝 Markdown 格式要求

为确保工具正常解析，建议导入的 Markdown 文档遵循以下格式：

```markdown
# 测试类别标题

## 测试点名称

### 用例编号
- **优先级**：关键/高/中/低
- **前置条件**：...
- **测试步骤**：...
- **预期结果**：...
```

## 🔧 功能按钮说明

| 按钮 | 功能说明 |
|------|----------|
| 📁 导入文件 | 选择并导入本地 Markdown 文件 |
| 🔄 刷新 | 刷新当前文档显示 |
| 📤 导出带评审 | 导出包含评审意见的完整文档 |
| 📄 导出无评审 | 导出原始测试用例文档 |

## 📊 统计面板说明

工具顶部提供实时统计信息：
- **测试类别**：文档中包含的测试类别总数
- **详细测试点**：所有测试点的总数量
- **关键/高/中/低**：各优先级测试用例的数量
- **评审意见**：已添加的评审意见总数
- **标签页**：使用的标签数量

## 💡 使用建议

1. **本地使用**：该工具完全在浏览器本地运行，无需联网，数据安全
2. **及时保存**：评审完成后及时导出，避免浏览器刷新导致数据丢失
3. **格式规范**：导入文档前确保 Markdown 格式符合要求
4. **定期导出**：建议定期导出备份评审结果

## ⚠️ 注意事项

- 工具数据存储在浏览器内存中，刷新页面将清空所有数据
- 请确保浏览器支持现代 HTML5 特性
- 大型文档可能影响加载速度，建议合理拆分文档
